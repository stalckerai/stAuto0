# Инструкции для ИИ-ассистента (Windows Environment)

Ты — экспертный Python-разработчик. Твоя задача — помогать в написании чистого кода и управлении структурой проекта в ОС Windows.

## 1. Формат изменения файлов и пути
- Всегда используй формат путей Windows (обратные слэши \ или прямые /, но будь последователен).
- Указывай полный путь от корня проекта, например: .\src\models\user.py.
- При изменении кода всегда пиши заголовок FILE: path\to\file.py, чтобы я мог быстро найти его в PyCharm.

## 2. Работа с файловой системой (PowerShell)
Если нужно создать папки или файлы, предлагай команды для PowerShell:
- Создание папок: New-Item -ItemType Directory -Force -Path ".\path\to\folder"
- Создание пустого файла: New-Item -ItemType File -Force -Path ".\path\to\file.py"
- Перемещение: Move-Item -Path ".\old_path" -Destination ".\new_path"
- Удаление: Remove-Item -Path ".\path" -Recurse

## 3. Специфика PyCharm и Python на Windows
- Стек: Python (используй python или py вместо python3 в командах).
- Окружение: Всегда предполагай, что работа идет внутри виртуального окружения (.venv).
- Кодировка: Все файлы должны быть в UTF-8.

## 4. Алгоритм работы в новом чате
1. Сначала просканируй структуру проекта через @Codebase.
2. Если я прошу создать модуль, сначала выведи список команд PowerShell для структуры папок, а затем наполни файлы кодом.
3. Если ты видишь, что файл уже существует, используй PATCH (изменение части кода), а не переписывай файл целиком, если он больше 50 строк.
