## Доработки

### Формат именования аккаунтов
Имена аккаунтов используют 3-значный padding: `auto_001`, `auto_002`, ..., `auto_010`, `auto_011`, ..., `auto_100`, ...

### Продолжение нумерации
При запуске скрипт читает `config/accounts.py`, находит максимальный существующий индекс и продолжает нумерацию.

Пример: если в `accounts.py` уже есть аккаунты `auto_001`...`auto_009` и `NUM_ACCOUNTS = 5`, то будут созданы `auto_010`, `auto_011`, `auto_012`, `auto_013`, `auto_014`.

### Сохранение существующих данных
- Существующие аккаунты из `accounts.py` сохраняются, новые добавляются в конец кортежа
- Существующие сиды из `auto_sids.py` сохраняются, новые добавляются
- Директории `config/chrome_accounts/auto_XXX` создаются только для новых аккаунтов

### Порты
`debugging_port` рассчитывается как `9330 + индекс` (auto_001 → 9331, auto_010 → 9340)
